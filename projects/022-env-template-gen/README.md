# Env Template Generator

A command-line tool that scans your codebase for `process.env.VAR` usage and automatically generates a `.env.template` file from the detected environment variables. This helps developers keep their environment templates synchronized with the actual code, preventing configuration drift in collaborative projects.

## Features

-   Scans specified directories and file patterns for environment variable usage.
-   Uses glob patterns for flexible file and directory inclusion/exclusion.
-   Parses `process.env.VAR`, `process.env['VAR']`, and `process.env["VAR"]` syntaxes.
-   Generates a `.env.template` file with all discovered variables.
-   Provides command-line options to specify source directories, output file path, and ignored patterns.
-   Gracefully handles read errors and parsing issues without crashing.
-   Appends a helpful header comment to the generated template file.

## Installation

You can use this tool by installing it globally via npm or by cloning the repository and running it locally.

**Global Installation (Recommended)**

```bash
npm install -g env-template-generator
```

**Local Installation (via Git Clone)**

```bash
# 1. Clone the repository
git clone https://github.com/your-username/env-template-generator.git

# 2. Navigate into the project directory
cd env-template-generator

# 3. Install dependencies
npm install

# 4. Run the tool using npm scripts or by linking it
npm link
# or
node bin/cli.js [options]
```

## Usage

The tool provides two binary names: `env-template-generator` and a shorter alias `gen-env-template`.

Once installed, you can run it from the root of your project. By default, it scans `src/**/*.{js,jsx,ts,tsx}` and creates a `.env.template` file in the current directory.

**Basic Command**

```bash
# Scans the default 'src' directory and generates .env.template
gen-env-template
```

**Command-Line Options**

```bash
$ gen-env-template --help

Usage: env-template-generator [options] [patterns...]

Scan for process.env usage and generate a .env.template file.

Options:
      --output, -o    Path to the output template file.
                               [string] [default: ".env.template"]
      --ignore, -i    One or more glob patterns to ignore.
                      [array] [default: ["node_modules/**","dist/**","build/**",
                               "coverage/**","**/*.test.js","**/*.spec.js"]]
      --verbose, -v   Enable verbose logging for debugging.
                                                      [boolean] [default: false]
  -h, --help          Show help                                        [boolean]
  -V, --version       Show version number                              [boolean]
```

## Examples

Here are a few examples of how to use the tool in different scenarios.

### Example 1: Default Scan

Run the tool with no arguments to scan the default `src` directory and generate a `.env.template` file in your project root.

**Command:**

```bash
gen-env-template
```

**Console Output:**

```
Starting environment template generation...

🔍 Scanning for source files...
Found 5 files to scan.

⚙️  Parsing files for environment variables...
Discovered 3 unique environment variables.

📝 Writing template file to: .env.template

✅ Success! Template file generated at '.env.template'.
```

**Generated `.env.template`:**

```
# This file was auto-generated by env-template-generator.
# It contains a list of all environment variables used by the application.
#
# Populate this file with your environment-specific values and rename it to .env
# for local development. DO NOT commit the .env file to version control.
#
# Generated on: 2023-10-27T10:00:00.000Z

API_KEY=
DATABASE_URL=
PORT=
```

### Example 2: Custom Source Directories and Output File

Scan specific directories (`lib` and `api`) and save the output to a different file named `.env.example`.

**Command:**

```bash
gen-env-template "lib/**/*.js" "api/**/*.ts" -o .env.example
```

**Console Output:**

```
Starting environment template generation...

🔍 Scanning for source files...
Found 8 files to scan.

⚙️  Parsing files for environment variables...
Discovered 5 unique environment variables.

📝 Writing template file to: .env.example

✅ Success! Template file generated at '.env.example'.
```

### Example 3: Ignoring Specific Files and Verbose Output

Scan the entire project for JavaScript files, but ignore a specific `config` directory. Use the `--verbose` flag to see detailed logs.

**Command:**

```bash
gen-env-template "**/*.js" --ignore "**/config/**" --verbose
```

**Console Output:**

```
Starting environment template generation...
  - Source Patterns: **/*.js
  - Ignore Patterns: **/config/**,node_modules/**,dist/**,build/**,coverage/**,**/*.test.js,**/*.spec.js
  - Output File: .env.template

🔍 Scanning for source files...
  - Found 2 files to scan.
    - /path/to/project/src/db.js
    - /path/to/project/src/server.js

⚙️  Parsing files for environment variables...
Discovered 2 unique environment variables.
  - Variables: DB_HOST, DB_USER

📝 Writing template file to: .env.template

✅ Success! Template file generated at '.env.template'.
```
